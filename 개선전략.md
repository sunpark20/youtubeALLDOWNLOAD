### 2. 성능 최적화

#### 현재 문제점
- 대용량 재생목록 다운로드 시 메모리 사용량 증가
- 아카이브 동기화가 매번 전체 파일 스캔

#### 개선 방안
- [ ] **아카이브 동기화 최적화**
  - 위치: utils.py:90-122
  - 방법: 파일 수정 시간 기반 캐싱
  - 예상 효과: 50% 이상 속도 향상

- [ ] **병렬 다운로드 옵션**
  - 현재: 순차 다운로드
  - 개선: yt-dlp `-N` 옵션으로 병렬 다운로드
  - 주의: CPU/네트워크 부하 증가

- [ ] **메모리 사용량 최적화**
  - 스트림 로그 버퍼링
  - 오래된 로그 자동 정리

### 4. 코드 품질 개선

#### 리팩토링 필요 영역

**app.py (11KB)**
- 현재: 모든 UI 로직이 main() 함수에 집중
- 개선: 탭별로 함수 분리
  ```python
  def render_full_channel_tab():
  def render_playlists_tab():
  def render_individual_playlist_tab():
  def render_single_video_tab():
  ```

**downloader.py (8.7KB)**
- 현재: run_download_process가 너무 복잡 (87줄)
- 개선: 옵션 빌더 패턴 도입
  ```python
  class DownloadOptionsBuilder:
      def with_format(self, format_str):
      def with_archive(self, path):
      def build(self):
  ```

**utils.py (6.3KB)**
- 현재: 설정, 바이너리, 아카이브 관리가 한 파일에
- 개선: 책임 분리
  - `config.py` - 설정 관리
  - `binary.py` - yt-dlp/ffmpeg 관리
  - `archive.py` - 다운로드 이력 관리
