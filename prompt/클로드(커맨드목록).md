# 커스텀 슬래시 커맨드 & 스킬 목록

## 커맨드 (`.claude/commands/`)

사용자가 직접 실행하는 슬래시 커맨드.

### /project:c-github-build

- **파일**: `.claude/commands/c-github-build.md`
- **요약**: GitHub Actions 빌드 & 릴리즈 (인텔맥/실리콘맥/Windows). 러너, entitlements, 태그 관리, 버전 올리기, 빌드 실패 대처법 포함
- **실행**: `/project:c-github-build` 또는 `/project:c-github-build v1.3.0 빌드해줘`
- **용도**: 새 버전 빌드 & 릴리즈할 때

### /project:c-crashreport

- **파일**: `.claude/commands/c-crashreport.md`
- **요약**: Gmail에서 `[CrashReport]` 이메일을 MCP로 읽어 에러 유형·버전·사용자·플랫폼별로 분석 보고. 마지막 읽은 날짜를 기억해서 다음 실행 시 새 리포트만 조회
- **실행**: `/project:c-crashreport` 또는 `/project:c-crashreport 최근 1.2.9 크래시만 봐줘`
- **용도**: 사용자 크래시 리포트 모니터링 및 분석

### /project:c-log-debug

- **파일**: `.claude/commands/c-log-debug.md`
- **요약**: 오늘 날짜의 로그 파일을 찾아 ERROR/WARNING 분석. macOS/Windows/레포 루트 로그 모두 지원
- **실행**: `/project:c-log-debug` 또는 `/project:c-log-debug 윈도우 로그 봐줘`
- **용도**: 다운로드 후 에러 원인 파악할 때

## 스킬 (`.claude/skills/`)

Claude가 자동으로 트리거하는 스킬. 사용자가 직접 실행하지 않음.

### post-command-improve

- **파일**: `.claude/skills/post-command-improve/SKILL.md`
- **요약**: `/project:` 커맨드 실행 완료 후 자동 트리거 → 결과 분석 → 코드 개선 → 커맨드 md 파일 다듬기
- **트리거**: 위 커맨드 중 하나가 실행 완료되면 자동 실행
