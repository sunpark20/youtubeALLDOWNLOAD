# 🛠️ YouTube Downloader 개발 지침 (DEV_GUIDE.md)

이 파일은 개발 시 **절대적으로 준수해야 하는 규칙**을 정의합니다.
코드를 수정하기 전에 반드시 이 내용을 확인하고 따르세요.

## 1. 핵심 원칙
- **단순함 유지:** 화려한 그래픽보다 정확한 동작에 집중한다.
- **주석 최소화:** 사용자에게 설명하는 주석은 적지 않는다. (코드를 깔끔하게 유지)
- **성능 최적화:** 로컬에서 빠르게 검사하고, 필요한 영상만 정밀 타격하여 다운로드한다.

## 2. 파일 구조 및 역할
- **`app.py`**: 웹 화면(Streamlit) 구성 및 사용자 입력 전달.
- **`downloader.py`**: 다운로드 엔진, 로그 파싱, 실시간 상태 업데이트.
- **`utils.py`**: 설정 로드/저장, 시스템 환경 확인.

## 3. 기능 작동 방식
- **다운로드 모드 (탭 순서 및 명칭 준수):**
    1. **전체 동영상 다운로드** (채널 내 모든 영상)
    2. **전체 재생목록 동영상** (채널 내 모든 재생목록을 폴더별로)
    3. **개별 플레이 리스트** (특정 재생목록 URL 하나만)
    4. **단일 동영상** (영상 하나만)

- **입력 규칙:**
    - 채널 핸들 (@는 제외하고 입력)

- **파일명 규칙 (중요):**
    - 순서 번호(01, 02...) 대신 **업로드 날짜**를 사용하여 불변성을 확보한다.
    - 포맷: `%(upload_date)s - %(title)s [%(id)s].%(ext)s`
    - (예: `20231225 - 영상제목 [AbCdEfGh].mp4`)

- **아카이브 관리:**
    - 모든 기록은 각 다운로드 폴더 내의 `.download_archive.txt` 파일에 개별 관리 (로컬 아카이브).
    - 다운로드 시작 전, 실제 파일과 아카이브를 대조하여 없는 파일은 기록에서 삭제(Sync).

- **UI 필수 정보:**
    - 현재 영상 순서 (예: 103 / 300).
    - 개별 파일 진행률 및 남은 용량.
    - 전체 파일 누적 용량 및 예상 총 용량.

## 4. 기타 주의 사항
- `yt-dlp` 실행 시 불필요한 메타데이터 파일 생성을 끈다.
- 쇼츠(Shorts)는 기본적으로 포함하지 않도록 설정한다.
- 사용자가 경로를 쉽게 선택할 수 있도록 폴더 탐색기 기능을 유지한다.
- 개별 동영상 다운로드 시에는 아카이브를 무시하고 강제로 다운로드한다.
