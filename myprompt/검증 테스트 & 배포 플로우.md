# 검증 테스트 & 배포 플로우

매 버전 릴리즈마다 반복하는 작업 정리.

---

## 1단계: 로컬 테스트 (맥북 실리콘, ./run_app.sh)

### 테스트 항목
- [ ] @채널명 URL → MP3 다운로드 (1~2개만 받고 중단)
- [ ] @채널목록 URL → MP3 다운로드 (1~2개만 받고 중단)
- [ ] @개별동영상 URL → MP3 다운로드
- [ ] 중복 체크: 위에서 중단한 것 다시 시작 → 이미 받은 파일 스킵되는지 확인
- [ ] 해당 버전의 신규 기능/수정사항 동작 확인

### 클로드가 대신 해줄 수 있는 것
- 코드 변경사항 diff 검증 (계획대로 적용됐는지)
- 코드 로직 흐름 추적 / 회귀 위험 분석
- build.yml 릴리즈 노트 내용·문법 확인
- `run_app.sh` 실행 전 의존성·설정 문제 사전 점검

### 내가 직접 해야 하는 것
- `./run_app.sh` 실행 후 브라우저에서 UI 동작 확인 (눈으로 봐야 함)
- 실제 다운로드·변환·중단·중복스킵 동작 확인
- 진행률 표시, 파일크기 표시 등 실시간 UI 확인

---

## 2단계: GitHub Actions 빌드

### 작업
- [ ] 커밋 & 푸시
- [ ] 태그 생성 & 푸시 → 빌드 자동 트리거
- [ ] 빌드 완료 대기

### 클로드가 대신 해줄 수 있는 것
- 커밋·태그·푸시 명령 실행
- `gh run watch`로 빌드 상태 모니터링
- 빌드 실패 시 로그 확인 & 원인 분석
- 빌드 실패 → 코드 수정 → 태그 재생성 → 재빌드 사이클

### 내가 직접 해야 하는 것
- 없음 (이 단계는 클로드가 전부 가능)

---

## 3단계: 빌드 결과물 테스트

### 테스트 항목 (1단계와 동일한 테스트를 빌드된 앱으로 반복)
- [ ] **macOS 실리콘**: 빌드된 .dmg 설치 → 1단계 테스트 반복
- [ ] **Windows**: 빌드된 .zip 설치 → 1단계 테스트 반복

### 클로드가 대신 해줄 수 있는 것
- GitHub Release에서 빌드 아티팩트 다운로드 링크 확인
- 릴리즈 페이지 내용 검증

### 내가 직접 해야 하는 것
- macOS / Windows 각각에서 빌드된 앱 설치 & 실행
- 1단계와 동일한 테스트 수동 진행
- Windows 전용 동작 확인 (숨김파일 등)

---

## 4단계: 배포 공지

### 작업
- [ ] 사용자들에게 업데이트 공지

### 클로드가 대신 해줄 수 있는 것
- 공지 문구 초안 작성

### 내가 직접 해야 하는 것
- 실제 공지 발송 (어디에 올릴지는 내가 결정)
