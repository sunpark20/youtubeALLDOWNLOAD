# 빌드할 때 클로드에게 시킬 것

## macOS 인텔 빌드 러너 주의사항

| 러너 | 아키텍처 | 무료 | 상태 |
|------|----------|------|------|
| `macos-13` | x86_64 | - | 2025-12 종료 |
| `macos-15-large` | x86_64 | **유료** | 결제 필요, 사용 금지 |
| `macos-15-intel` | x86_64 | **무료** | 현재 사용 중, 2027-08까지 지원 |
| `macos-14`, `macos-15` | arm64 | 무료 | 인텔 빌드 불가 |

- **인텔 빌드 러너는 반드시 `macos-15-intel` 사용** (`macos-15-large`는 유료라 결제 오류 발생)
- arm64 러너(macos-14, macos-15)에서 Rosetta 2로 x86_64 빌드: GitHub Actions에서 **불가**
- Universal2 빌드: PyInstaller에서 **비실용적** (모든 의존성이 universal2여야 함)

## 태그 재생성으로 빌드 재실행하는 방법

```bash
# 1. 태그 삭제 (로컬 + 리모트)
git tag -d v1.x.x
git push origin :refs/tags/v1.x.x

# 2. 태그 재생성 + 푸시 (빌드 자동 트리거)
git tag v1.x.x
git push origin v1.x.x
```

## 인텔맥 entitlements 필수 권한 (v1.2.4에서 해결됨)

PyInstaller + Hardened Runtime 조합에서 인텔맥은 아래 2개 entitlement가 없으면 앱 실행 자체가 안 됨 (Python 로드 전 OS가 kill).
v1.2.4에서 추가하여 해결 완료 — 사용자 테스트 확인됨.

```xml
<key>com.apple.security.cs.allow-unsigned-executable-memory</key>
<true/>
<key>com.apple.security.cs.disable-library-validation</key>
<true/>
```

- **원인**: PyInstaller가 실행 시 `.so`/`.dylib`를 임시 디렉토리에 추출 후 동적 로드 → 인텔맥에서는 unsigned library 로드가 차단됨
- **Apple Silicon은 왜 되나**: ARM64 메모리 보안 모델이 달라서 이 권한 없이도 동작
- **파일**: `entitlements.plist`
- **절대 삭제 금지**
